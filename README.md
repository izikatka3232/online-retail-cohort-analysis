# Online Retail Cohort Analysis

Анализ удержания клиентов с помощью Cohort Analysis на примере Online Retail II датасета.

Этот проект демонстрирует применение Cohort Analysis — анализ поведения групп клиентов, объединённых по времени их первой покупки.

## Что это?

Проект отвечает на ключевые вопросы:

- Сколько % клиентов возвращаются в каждый месяц после первой покупки?
- Когда происходит максимальное выбывание клиентов?
- Какую ценность (LTV) генерирует каждая когорта за время жизни?
- Как изменяется качество привлекаемых клиентов?

## Ключевые выводы

### 1. Критический первый месяц — 79% потеря retention

Максимальный отток происходит именно в первый месяц. Это типично для e-commerce — большая часть первых покупателей — это "проверяющие" или случайные посетители.

| Период | Retention | Интерпретация |
|--------|-----------|---------------|
| M+0 | 100% | Базовый месяц (все покупатели) |
| M+1 | 21.2% | 79% клиентов не возвращаются |
| M+3 | 21.6% | Стабилизация |
| M+6 | 17.8% | Плато |
| M+12 | 18.2% | Долгосрочное ядро |

### 2. Низкая retention, но растущий LTV

Несмотря на 79% падение retention, оставшиеся клиенты генерируют значительную стоимость.

Оставшиеся 21% клиентов — это наиболее ценные и лояльные сегменты. Они составляют основу долгосрочной выручки.

| Период | LTV за период | Интерпретация |
|--------|----------------|---------------|
| M+0 | $508.31 | Первая покупка |
| M+1 | $104.69 | Повторные покупки |
| M+12 | $150.25 | Долгосрочная стоимость |

### 3. После месяца 3 — стабилизация

```
M+0 → M+1:    Максимальный drop (-79%)
M+1 → M+3:    Медленная стабилизация (-0.4%)
M+3 → M+12:   Плато (изменения минимальны)
```

Если клиент пережил первый месяц и вернулся, он с высокой вероятностью останется в течение 12+ месяцев.

### 4. Качество когорт со временем улучшается/ухудшается

```
Старые когорты (2010-12 — 2011-01):  M+1 = 100%
Новые когорты (2011-11 — 2011-12):   M+1 = 100%

Тренд по LTV: Новые когорты показывают сопоставимые значения
```

Бизнес привлекает примерно одинаковые по качеству когорты клиентов на протяжении периода.

## Вопрос 1: "Почему мы должны верить этим выводам? Как вы знаете, что это сработает?"

### Суть возражения
Менеджеры сомневаются в выводах анализа:
- "Может, 79% drop это нормально для e-commerce?"
- "Может, наши клиенты совсем другие?"
- "Это может не сработать в нашем случае"

---

### Решение 1: Валидировать на собственных данных

**Что делать:**
1. Запустить cohort analysis на собственном датасете компании (не на примере)
2. Сравнить результаты с примером из проекта
3. Если результаты похожи (70-85% drop в M+0→M+1) → выводы применимы

**Почему это работает:**
- Менеджеры видят "реальные числа" своей компании
- Если drop похож на пример (79% vs ваш 75%) → выводы валидны
- Если drop другой (например, 40%) → может быть другая проблема

**Пример скрипта (на Python):**
```python
# Рассчитать retention по вашим данным за последние 12 месяцев
# Результат покажет, что drop в M+1 действительно есть
# И скорее всего он в диапазоне 70-85%
```

**Таймлайн:** 3-5 дней анализа (1 аналитик)  
**Результат:** Доказательство, что проблема реальная  

---

### Решение 2: Провести A/B тест на малой части трафика

**Что делать:**
1. Выбрать один из рекомендуемых действий (например, email sequence)
2. Тестировать на 10% новых клиентов (контрольная группа)
3. Дать 2-4 недели на результаты
4. Сравнить M+1 retention между группами

**Почему это работает:**
- "Пусть данные сами расскажут" — самое убедительное
- Малый риск (только 10% трафика)
- Быстрый результат (2-4 недели, а не полгода)

**Пример результата:**
```
Контрольная группа (обычное): 21% M+1 retention
Тестовая группа (с email):    26% M+1 retention
Улучшение: +5% → доказано!

На 10,000 клиентов это = 500 дополнительных клиентов
На $150 LTV это = $75,000 дополнительного дохода
```

**Таймлайн:** 4-6 недель (2 недели подготовки + 2-4 недели тестирования)  
**Стоимость:** $2,000-5,000 (setup + анализ)  
**Результат:** Убедительное доказательство  

---

### Решение 3: Сравнить с индустриальными бенчмарками

**Что делать:**
1. Найти публичные данные о retention в e-commerce
2. Сравнить с вашими результатами
3. Показать, что 79% drop не аномалия, а стандарт

**Пример данных (индустрия):**
```
E-commerce retention benchmarks:
- Месяц 1 (M+1): 20-25% для большинства компаний
- Месяц 3 (M+3): 18-22%
- Месяц 12 (M+12): 15-20%

Ваши данные: M+1 = 21.2% → в норме
Но это значит, что 79% drop это СТАНДАРТНАЯ ПРОБЛЕМА

Пример компаний с better retention:
- Amazon: ~40-50% M+1 (потому что ecosystem)
- Warby Parker: ~35-40% M+1 (потому что качество, loyalty program)
- Средний e-commerce: 20-25% M+1 → как у вас

Вывод: Есть потенциал поднять с 21% до 35-40% (как лучшие)
```

**Источники:**
- Reforge (платформа для маркетеров) — публичные данные
- Statista — e-commerce benchmarks
- SaaS метрики (похоже на e-commerce по паттернам)

**Таймлайн:** 2-3 дня исследования  
**Стоимость:** Бесплатно (публичные данные)  
**Результат:** Контекст: "это не наша уникальная проблема, это индустрия"

## Вопрос 3: "Скидки будут убийственны для маржи! Мы потеряем прибыль"

### Суть возражения
Финансовый или операционный директор:
- "Если мы даём 15% скидку на второй заказ, мы теряем 15% от этого заказа"
- "При марже 30%, это дыра в бюджете"
- "Скидки это порочный круг — если дать один раз, клиент всегда ждёт"

---

### Решение 1: Показать, что скидка окупается в 3+ раз через LTV

**Что делать:**
Рассчитать экономику клиента с и без скидки.

**Сценарий 1: БЕЗ скидки (текущее)**
```
M+0:
- Средний заказ:         $508
- Маржа 30%:            $152
- Вероятность M+1:      21.2%

M+1:
- Вероятность заказа:   21.2% (из 100 клиентов только 21 вернулись)
- Средний заказ:        $105 (меньше, потому что повторный)
- Маржа 30%:            $31.50
- LTV за 12 месяцев:    $150 × 30% = $45 маржи

Полный LTV за клиента: $152 (M+0) + $45 (M+1-M+12) = $197 маржи
```

**Сценарий 2: С скидкой 15% на M+1 (рекомендация)**
```
M+0:
- Средний заказ:         $508
- Маржа 30%:            $152

M+1 (с 15% скидкой):
- Вероятность заказа:   30% (скидка привлекает!) вместо 21.2%
- Средний заказ:        $105 × (1-0.15) = $89.25
- Маржа 30% от $89:     $26.70
- LTV за 12 месяцев:    $175 × 30% = $52.50 маржи

Полный LTV за клиента: $152 (M+0) + $52.50 (M+1-M+12) = $204.50 маржи
```

**Сравнение:**
```
Без скидки:   $197 маржи на клиента
С скидкой:    $204.50 маржи на клиента
Разница:      +$7.50 маржи (3.8% увеличение)

ВЫВОД: Скидка 15% окупается за счёт того, что:
1. +8.8% больше клиентов вернулось (30% vs 21.2%)
2. Маржа на повторный заказ выше, чем потеря на скидку
3. LTV растёт несмотря на скидку!
```

**Масштабирование:**
```
На 100,000 клиентов в год:
Без скидки:   100,000 × $197 = $19,700,000 маржи
С скидкой:    100,000 × $204.50 = $20,450,000 маржи
Дополнительно: $750,000 маржи (даже с 15% скидкой!)
```

---

### Решение 2: Ограничить скидку (не давать её всем и не давать часто)

**Что делать:**
Использовать скидку стратегически, не везде.

**Вариант 1: Скидка только для "рискованных" клиентов**
```
Дают скидку только если:
- Первый заказ был < $100 (низкий намерение)
- ИЛИ первый заказ был более 45 дней назад
- ИЛИ тип товара указывает на low engagement

Остальные 40%+ не получают скидку → не теряем маржу на них
Только 25-30% получают скидку → экономим на скидковой части
```

**Вариант 2: Взамен скидке — добавить стоимость**
```
Вместо скидки 15%:
- Дать бесплатную доставку (экономия $5-10)
- Дать бесплатный мини-товар стоимостью $3-5
- Дать баллы лояльности (использоваться позже)

Психологический эффект одинаков, но маржа меньше теряется
```

**Вариант 3: Скидка по времени**
```
Скидка 15% действует только в первые 2 недели
Потом уходит, но уже сформирована привычка покупать

Клиент получит привычку к бренду раньше, чем привычку к скидке
```

---

### Решение 3: Альтернатива скидкам — увеличить AOV вместо скидок

**Что делать:**
Вместо скидок, увеличивать Average Order Value (AOV) рекомендациями.

**Без скидок:**
```
M+0: $508 заказ → маржа $152
M+1: $105 заказ → маржа $31.50

Проблема: второй заказ мал, маржа низка
```

**С рекомендациями AOV (без скидок):**
```
M+0: $508 заказ → маржа $152
M+1: 
  - Рекомендовать товар к товару (аксессуары, расходники)
  - Коробка товаров (подборка)
  - Бандлы (2 товара дешевле, чем по отдельности)
  - Результат: AOV растит до $140-160 вместо $105
  
M+1: $150 заказ → маржа $45 (больше чем с скидкой!)
Дополнительно: если AOV растит, M+1 retention улучшается 
  (больше вовлечение = выше retention)
```

**Сравнение всех трёх вариантов:**
```
Вариант 1 (скидка 15%):
- M+1 заказ: $89.25
- M+1 маржа: $26.70
- M+1 retention: 30%

Вариант 2 (скидка + ограничение):
- M+1 заказ: $100
- M+1 маржа: $28
- M+1 retention: 27%

Вариант 3 (AOV через рекомендации):
- M+1 заказ: $155
- M+1 маржа: $46.50
- M+1 retention: 32% (выше, чем со скидкой!)
```

**Вывод:** Вариант 3 (рекомендации) лучше всех!
- Не теряем маржу
- Даже увеличиваем AOV
- Retention выше
- Клиент не привыкает к скидкам

## Bonus: Вопрос, который может появиться в год 2

### "Почему результаты остановились? Мы внедрили ВСЁ, но результаты не растут дальше"

**Ожидаемая ситуация (месяц 7-12):**
```
Месяцы 1-3:  Быстрый рост (от 21% к 35%)
Месяцы 4-6:  Медленнее (от 35% к 38%)
Месяцы 7-12: Плато (остаём на 38%)
```

**Почему это нормально:**
```
Каждое действие (email, скидка, AOV) имеет максимальный потенциал:
- Email: может улучшить максимум на 8-10%
- Скидка: максимум 6-8%
- AOV рекомендации: максимум 5-7%
- Support: максимум 3-4%

Сумма всех максимумов: 20-30%
Поэтому плато на 38-40% (21% baseline + 17-19% improvements) это ОЖИДАЕМО
```

**Решение:**
```
Для дальнейшего роста нужны новые инициативы:
1. RFM сегментация (более целевые действия)
2. Predictive модель (кто уйдёт в M+2, предупредить)
3. Product-level анализ (какие товары удерживают лучше)
4. Реферальная программа (вовлечь текущих клиентов)
5. Subscription модель (если применимо)

Это уже следующий уровень оптимизации (год 2)
```

| Возражение | Решение | ROI | Таймлайн |

| "Почему верить?" | A/B тест на 10% трафика | 11x | 4-6 недель |
